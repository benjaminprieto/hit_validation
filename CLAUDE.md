# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

A modular pipeline (Python) for **rigorous validation of virtual screening hits** using DOCK6 with footprint-based interaction analysis and GB/SA implicit solvation. Evaluates each hit on its own absolute merits to produce evidence for Pharmit template selection and purchase decisions. Pre-computed reference data (e.g., UDX from `reference_docking`) can be imported as informational context in output tables, but is never used as input to any pipeline scoring step. Written in Spanish (README, comments) by Benjamin Prieto / Environ Bio.

### The Iterative Drug Discovery Cycle

```
reference_docking → molecular_docking → HIT_VALIDATION → Pharmit → reference_docking [iteration 2] → ...
```

- **`reference_docking`**: Establishes the crystallographic reference baseline (UDX in iteration 1, best hit in iteration 2+)
- **`molecular_docking`**: High-throughput virtual screening against molecule libraries
- **`hit_validation`** (THIS PROJECT): Rigorous validation of screening hits — per-molecule evidence for Pharmit template or purchase

### Fundamental Differences vs reference_docking

| Aspect | reference_docking | hit_validation |
|--------|-------------------|----------------|
| Subject | Single crystallographic ligand (UDX) | Multiple screening hits (10-20 molecules) |
| Coordinates | Crystal coordinates from PDB HETATM | Generated by docking (no crystal pose) |
| Ligand prep | No antechamber (preserves crystal geometry) | Antechamber required (AM1-BCC + Sybyl) |
| Box margin | 1.0 A (tight, keeps ligand near crystal pose) | 5.0 A (exploration space for hits) |
| Docking method | Rigid (crystal pose stays put) | Flex (anchor-and-grow conformational search) |
| PLIP input | Crystal complex (receptor + co-crystallized UDX) | Docked complex (receptor + best docked pose) |
| GB/SA minimize | false (crystal geometry is sacred) | true (docked poses benefit from relaxation) |
| Reference ligand | UDX IS the subject | None (hits evaluated on own merits; UDX data imported as context in outputs) |
| Output | Reference energy profile for comparison | Per-molecule validation report with purchase arguments |
| 06a/06b | Pharmacophore from crystal UDX | Pharmacophore from best validated hit (iteration 2) |

## Module-by-Module Specification

### 00a — Ligand Preparation (REWRITE from reference_docking)

Prepares N screening hit ligands using antechamber with AM1-BCC charges + Sybyl atom types.

**Core function:**
```python
def run_ligand_preparation(
    ligand_dir: Union[str, Path],   # Directory with input mol2/SDF files
    output_dir: Union[str, Path],
    atom_type: str = "sybyl",       # MUST be sybyl for DOCK6
    charge_method: str = "bcc",     # AM1-BCC
    docking_ph: float = 6.3,
    timeout_per_molecule: int = 300,
    molecule_filter: Optional[List[str]] = None,
) -> Dict[str, Any]
```

**3-tier fallback:**
1. `antechamber -c bcc -at sybyl` (full AM1-BCC + Sybyl)
2. `antechamber -c gas -at sybyl` (Gasteiger + Sybyl, fast, no QM — solves timeouts on 79+ atom molecules)
3. `obabel` conversion (last resort, warns about degraded atom typing)

**Pipeline per molecule:** SDF→mol2 conversion → pH protonation (ionization package) → antechamber → validation

**Config (`03_configs/00a_ligand_preparation.yaml`):**
- `atom_type: "sybyl"` — MUST be sybyl. GAFF2 causes DOCK6 to silently fall back to rigid docking.
- `charge_method: "bcc"` — AM1-BCC (gold standard)
- `timeout_per_molecule: 300`

**Output:** `{name}/{name}.mol2`, `preparation_status.csv`, `preparation_summary.txt`

### 00b — Receptor Preparation (IDENTICAL to reference_docking)

ChimeraX DockPrep → `rec_charged.mol2` with AMBER ff14SB charges and Sybyl atom types.

**Core function:** `run_receptor_preparation(receptor_pdb, output_dir, protonation_tool, chain, ph, ...)`

**Output:** `rec_charged.mol2`, `rec_noH.pdb`, `receptor_protonated.pdb`, `protonation_report.json`

**Optimization:** Campaign config can set `precomputed_mol2` to skip this step if receptor was already prepared in reference_docking.

### 00d — Binding Site Definition (skipped if precomputed grids)

Trims receptor PDB for grid generation. **Skipped entirely** if `precomputed_grids_dir` is set in campaign config (preferred workflow — reuse grids from reference_docking or molecular_docking).

**Core function:** `run_binding_site_definition(receptor_noH_pdb, output_dir, method, residue_ids=None, center=None, contact_cutoff=5.0, trim_radius=25.0)`

**Methods:** residues (default) | coordinates | reference_ligand (legacy)

**Output:** `rec_noH_site.pdb`, `binding_site_report.json`, `binding_site_summary.txt`

### 01b — Grid Generation (skipped if precomputed grids)

DMS surface → sphere generation → sphere selection → box → energy grids. **Skipped** if `precomputed_grids_dir` is set in campaign config.

**Core function:** `run_grid_generation(receptor_noH_pdb, receptor_charged_mol2, output_dir, box_margin=5.0, ...)`

**Config:** `box_margin: 5.0` (was 1.0 in reference_docking). Hits need conformational freedom.

**Output:** `receptor.ms`, `all_spheres.sph`, `spheres_ligand.sph`, `spheres_ligand_box.pdb`, `ligand.nrg`, `ligand.bmp`

### 01c — DOCK6 Docking (CONFIG ONLY — flex + clustering cutoff)

Runs DOCK6 docking with grid_score as primary scoring function.

**Core function:** `run_dock6_batch(ligand_mol2_dir, spheres_file, grid_prefix, output_dir, search_method="rigid", ...)`

**Config changes:**
- `search_method: "flex"` (was "rigid") — hits need anchor-and-grow conformational search
- `pruning_clustering_cutoff: 2.0` — **CRITICAL**: the DOCK6 default of 100 A merges ALL poses into 1 cluster, producing only 1 output pose
- `num_preclustered_conformers: 1000` (was 500) — more sampling for hits
- `timeout_per_molecule: 600`

**DOCK6 templates:** `DOCK6_FLEX_TEMPLATE` (23 params) and `DOCK6_RIGID_TEMPLATE`. Neither contains any `_secondary` score parameters.

**Output per molecule:** `dock6.in`, `dock6.out`, `{name}_scored.mol2`

### 01d — Footprint Rescore (self-reference mode)

Re-scores existing poses with per-residue vdW + ES decomposition using `footprint_similarity_score_primary`. Each hit uses its own scored pose as the reference (self-reference) — no UDX mol2 needed.

**Core function:** `run_footprint_rescore(docking_dir, output_dir, receptor_mol2, reference_mol2="self", dock6_home="/opt/dock6", ...)`

**Template:** `FPS_RESCORE_TEMPLATE` — 8 params. No `_secondary`, no `gbsa_hawkins`.

**Output per molecule:** `{name}_fps_scored.mol2`, `{name}_fps_footprint_scored.txt`, `{name}_fps_hbond_scored.txt`

### 01f — GB/SA Hawkins Rescore (CONFIG ONLY — minimize changed)

Implicit solvation scoring as separate rigid rescore with `gbsa_hawkins_score_primary=yes`.

**Core function:** `run_gbsa_rescore(docking_dir, output_dir, receptor_mol2, minimize=False, solvent_dielectric=78.5, salt_concentration=0.15, gb_offset=0.09, ...)`

**Config change:** `minimize: true` (was false). Docked poses benefit from simplex relaxation before GB/SA.

**Template:** `GBSA_RESCORE_TEMPLATE` — 11 params including `{minimize}` and `{minimize_params}`.

**Output per molecule:** `{name}_gbsa_scored.mol2`

### 01e — Score Collection (IDENTICAL to reference_docking)

Parses all DOCK6 scored mol2 files and collects scores into CSV/Excel.

**Core function:** `run_score_collection(docking_dir, output_dir, score_key="Grid_Score", max_molecules=0, extract_best_pose_mol2=True, keep_all_poses=False, ...)`

**Captured DOCK6 header fields:** Grid_Score, Grid_vdw_energy, Grid_es_energy, Internal_energy_repulsive, DOCK_Rotatable_Bonds, Formal_Charge, Heavy_Atoms, Molecular_Weight, HBond_Acceptors, HBond_Donors

**Output:** `dock6_scores.csv`, `dock6_scores.xlsx`, `best_poses/{name}.mol2`, `score_collection_report.json`

### 07a — Decision Report (NEW — does not exist in reference_docking)

Generates consolidated HTML report + CSV ranking with per-molecule validation evidence.

**Core function:**
```python
def run_decision_report(
    scores_csv: Union[str, Path],
    output_dir: Union[str, Path],
    plip_dir: Optional[str] = None,
    footprint_dir: Optional[str] = None,
    reference_scores_path: Optional[str] = None,

```

**Classification logic (absolute thresholds — not relative to UDX):**
- **"Strong Pharmit template candidate"**: covers >= 2 drug-like zones with meaningful energy (< -0.5 kcal/mol per zone)
- **"Purchase candidate"**: Grid Score <= -30.0 AND covers xylose OR uracil zone
- **"Weak candidate"**: otherwise

If `reference_context` is provided in campaign config, UDX appears as a reference row in all tables for human context — but does NOT participate in ranking or classification.

**KEY_RESIDUES** for detailed comparison: TRP392, HIS335, GLU529, ASP494, ARG598

**Config (`03_configs/07a_decision_report.yaml`):**
- `zone_energy_cutoff: -0.5` (absolute energy threshold for zone coverage)
- `min_zones_for_template: 2`
- `grid_score_purchase_threshold: -30.0`
- Reference paths auto-resolved from `campaign_config.reference_context`

**Output:** `decision_report.html` (one card per molecule), `decision_summary.csv`

## DOCK6.13 Scoring Architecture

DOCK6.13 silently ignores all `_secondary` scoring functions during flexible docking. Therefore:
- **01c** uses only `grid_score_primary` (docking step)
- **01d** uses `footprint_similarity_score_primary` (separate rescore)
- **01f** uses `gbsa_hawkins_score_primary` (separate rescore)

Each scoring function runs as an independent rigid rescore pass with `orient_ligand=no`.

**DOCK6 template strings** in dock6_runner.py, footprint_rescore.py, gbsa_rescore.py, footprint_rescoring.py contain **zero** `_secondary` parameters. This is verified by tests.

## Two-Layer Design

Each module has exactly two files:
1. **Core** (`01_src/hit_validation/module.py`): Pure logic, receives paths/dicts, returns results. All the science lives here.
2. **Script** (`02_scripts/module.py`): Thin CLI wrapper that merges YAML config + campaign_config + CLI overrides, then calls the core function.

When modifying behavior, edit the core module. When changing CLI args or config loading, edit the script.

`campaign_config.yaml` defines everything about a validation experiment: receptor PDB, ligand directory, pH, binding site, grid paths, reference data paths. Module YAMLs in `03_configs/` control only algorithm parameters.

## Key Domain Constraints

- **DOCK6 80-char path limit**: Fortran programs truncate paths. The pipeline uses symlinks in working directories — don't break this by hardcoding long absolute paths in DOCK6-related modules.
- **atom_type MUST be sybyl**: GAFF2 types cause DOCK6 to silently fall back to rigid docking with zero molecular descriptors. Module 00a enforces sybyl.
- **pruning_clustering_cutoff MUST be 2.0**: The DOCK6 default of 100 A merges ALL poses into 1 cluster, producing only 1 output despite requesting 100. Set in `03_configs/01c_dock6_run.yaml`.
- **Receptor mol2 must have AMBER ff14SB charges** and Sybyl atom types for DOCK6 compatibility. The `receptor_preparation.py` module handles this via ChimeraX.
- **No UDX mol2 in this pipeline**: UDX is never docked, scored, or analyzed here. Pre-computed UDX data from `reference_docking` can be imported via `reference_context` in campaign config as informational context in output tables. The pipeline works identically with or without it.
- **Binding site** is defined by residue list or coordinates (not by a reference ligand mol2). Alternatively, use `precomputed_grids_dir` to skip binding site definition + grid generation entirely.
- **PLIP protonates internally** — do NOT pre-protonate the ligand pose before feeding to PLIP.
